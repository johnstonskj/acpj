Provides pre-built services that can be used to monitor an application that has been built using actors, these services have been inspired by existing capabilities in the Erlang OTP platform. The package includes the ability to log messages (an Actor facade to Java logging) and a service to monitor and log actor birth and death. a

The purpose of the package is to provide a set of services that can be used by programmers to monitor their applications as the build and use actors.

What the rdf Package Contains

  • Services:
    • {@link com.googlecode.acpj.services.BasicService} -- The main framework class that implements the structure and run-loop for a service and allows implementors to focus on simply handling each request as the service provides it to them.
    • {@link com.googlecode.acpj.services.LogService} -- An actor-based facade over the standard Java logging, allows LogRecord instances to be sent to a common actor.
    • {@link com.googlecode.acpj.services.WatchdogService} -- A service that monitors the events generated when any actor is created, terminates correctly or dies in an unexpected way.
